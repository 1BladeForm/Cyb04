## Анализ модели OSI

### 1. Физический уровень (Physical Layer)
- **Обзор**: Этот уровень занимается физической передачей данных по сетевым каналам и устанавливает параметры для передачи битов, включая электрические, механические и процедурные характеристики.
- **Основные функции**:
  - Передача необработанных битов через физические носители (кабели, оптоволокно).
  - Определение характеристик сигналов (напряжение, частота).
  - Установка и разрыв соединений.
- **Примеры**: Ethernet-кабели, разъемы RJ45, радиосигналы Wi-Fi.
- **Типичные атаки**:
  - **Перехват сигнала**: Использование специального оборудования для перехвата данных, передаваемых через физические каналы.
  - **Физическое разрушение**: Повреждение или уничтожение кабелей и оборудования.
  - **Подмена устройств**: Использование поддельных устройств для подключения к сети.

### 2. Канальный уровень (Data Link Layer)
- **Обзор**: Этот уровень обеспечивает надежную передачу данных между узлами в одной сети и управляет доступом к физическому каналу.
- **Основные функции**:
  - Формирование кадров (фреймов) из битов.
  - Обнаружение и коррекция ошибок.
  - Управление доступом к среде передачи (например, CSMA/CD для Ethernet).
- **Примеры**: Ethernet, Wi-Fi (IEEE 802.11), протоколы PPP (Point-to-Point Protocol).
- **Типичные атаки**:
  - **ARP Spoofing**: Отправка поддельных ARP-сообщений для подмены MAC-адреса.
  - **MAC Flooding**: Заполнение таблицы MAC-адресов коммутатора для перехвата трафика.
  - **Атака Evil Twin**: Создание поддельной точки доступа Wi-Fi для перехвата данных.

### 3. Сетевой уровень (Network Layer)
- **Обзор**: Этот уровень отвечает за маршрутизацию пакетов данных между разными сетями, управляет логическими адресами и определяет пути передачи данных.
- **Основные функции**:
  - Определение маршрутов для передачи данных.
  - Логическая адресация (например, IP-адреса).
  - Фрагментация и сборка пакетов.
- **Примеры**: IP (Internet Protocol), ICMP (Internet Control Message Protocol), маршрутизаторы.
- **Типичные атаки**:
  - **DDoS-атака**: Атака, направленная на исчерпание ресурсов сети или сервера.
  - **IP Spoofing**: Отправка пакетов с подделанным IP-адресом.
  - **Ping of Death**: Отправка чрезмерно больших ICMP-пакетов для сбоя системы.

### 4. Транспортный уровень (Transport Layer)
- **Обзор**: Этот уровень обеспечивает надежную передачу данных между хостами, управляет потоком данных и устанавливает соединения.
- **Основные функции**:
  - Установление, управление и завершение соединений.
  - Обеспечение надежности (например, повторная передача потерянных пакетов).
  - Контроль потока и сегментация данных.
- **Примеры**: TCP (Transmission Control Protocol), UDP (User Datagram Protocol).
- **Типичные атаки**:
  - **TCP SYN Flood**: Атака с отправкой множества SYN-запросов, не завершая процесс установки соединения.
  - **Session Hijacking**: Перехват активной сессии пользователя.
  - **UDP Flood**: Отправка большого количества UDP-пакетов для перегрузки сети.

### 5. Сессионный уровень (Session Layer)
- **Обзор**: Этот уровень управляет сессиями и соединениями между приложениями, обеспечивая их синхронизацию и управление.
- **Основные функции**:
  - Установка, управление и завершение сессий.
  - Синхронизация взаимодействия между приложениями.
  - Обеспечение управления диалогом (полудуплексный или полный дуплекс).
- **Примеры**: RPC (Remote Procedure Call), NetBIOS.
- **Типичные атаки**:
  - **Session Fixation**: Принуждение пользователя использовать определенный идентификатор сессии.
  - **Man-in-the-Middle (MitM)**: Перехват и изменение данных между двумя сторонами.
  - **Replay Attack**: Повторное использование ранее захваченных сообщений.

### 6. Представительский уровень (Presentation Layer)
- **Обзор**: Этот уровень отвечает за преобразование данных для приложения, включая кодирование, шифрование и сжатие.
- **Основные функции**:
  - Преобразование форматов данных (например, ASCII в EBCDIC).
  - Шифрование и дешифрование данных.
  - Сжатие данных для уменьшения объема передаваемой информации.
- **Примеры**: JPEG, MPEG, SSL/TLS.
- **Типичные атаки**:
  - **SQL Injection**: Внедрение вредоносного SQL-кода через пользовательский ввод.
  - **Cross-Site Scripting (XSS)**: Внедрение вредоносного скрипта на веб-страницу.
  - **Data Manipulation**: Изменение данных при передаче.

### 7. Прикладной уровень (Application Layer)
- **Обзор**: Этот уровень предоставляет интерфейс для пользовательских приложений и сетевых сервисов, взаимодействующих непосредственно с пользователем.
- **Основные функции**:
  - Обеспечение сетевых услуг для приложений.
  - Обработка запросов и ответов от пользователей.
  - Поддержка различных сетевых приложений.
- **Примеры**: HTTP (Hypertext Transfer Protocol), FTP (File Transfer Protocol), SMTP (Simple Mail Transfer Protocol).
- **Типичные атаки**:
  - **Phishing**: Мошеннические попытки получения конфиденциальной информации.
  - **Malware**: Вредоносные программы, такие как вирусы и трояны.
  - **Denial of Service (DoS)**: Перегрузка приложения для его недоступности.
